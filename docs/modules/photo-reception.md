# 임상사진 무선 수신

## 1. 업무 배경

### 치과에서 임상사진이란

치과 진료에서 구강 내 사진은 진단·치료계획·경과관찰의 핵심 기록이다. DSLR 카메라로 촬영하며, 환자 한 명당 3~10장, 교정과의 경우 매 방문마다 세트로 촬영한다. 이 사진이 환자 차트에 정확히 붙어야 의미가 있다.

### 현재의 촬영-저장 파이프라인

카메라에서 PC까지의 전송은 EMR이 관여할 영역이 아니다. Eye-Fi, Camfi, 카메라 내장 Wi-Fi 등 다양한 전송 수단이 있지만, 최종 결과는 동일 — **PC의 특정 폴더에 이미지 파일이 생성되는 것**. EMR은 이 폴더만 감시하면 된다.

대부분의 치과(약 90%)는 카메라가 한 대이므로 감시 대상 폴더도 하나다. 다수의 진료실을 운영하는 치과(약 10%)에서만 카메라가 여러 대이고, 그에 따라 감시 폴더도 복수가 된다.

### 현행 시스템(덴트웹)의 문제

덴트웹에서는 사진을 EMR에 추가하려면 **수신대기를 수동으로 시작**해야 한다. 별도 모달 창을 열어서 "수신 시작" 버튼을 누른 뒤, 그 창 안에서 사진을 확인하고 환자에 추가한다.

바쁜 진료 중에 이 과정을 깜빡하면 사진이 폴더에만 존재하고 EMR에서는 인식하지 못한다. 나중에 파일 탐색기나 이미지 뷰어에서 수동으로 찾아 붙여야 하는데, 어떤 사진이 어느 환자 것인지 기억에 의존해야 한다.

근본 원인은 "폴더 감시"라는 단순한 작업을 사용자의 명시적 액션에 의존하게 설계한 것이다.

## 2. 실제 업무 흐름

### 일반과 진료실의 하루

원장이 환자를 진료한다. 진료 중 구강 사진을 찍어야 하는 시점이 오면, 어시스턴트가 DSLR을 건네거나 미러·리트랙터를 잡아준다. 원장이 촬영 후 카메라를 내려놓으면, Wi-Fi를 통해 사진이 PC 폴더에 도착한다. 전자차트에 사진을 추가하려고 보니 Eye-fi 버튼이 눌러져 있지 않다. 현행 시스템에서는 미리 버튼을 눌러놓지 않으면 수동으로 사진을 다시 추가해야 한다.

### 다중 카메라 환경 (소수 치과)

진료실이 2~3개인 치과에서는 각 진료실에 카메라가 있고, 각각 다른 폴더로 사진이 전송된다. "이 사진이 어느 진료실에서 온 건지"를 구분할 수 있으면 편하지만, 대다수의 치과에서는 카메라가 한 대이므로 이 구분이 필요 없다. 따라서 다중 소스 구분은 지원하되 UI에서 강조하지 않는다.

## 3. 문제와 설계 의도

### "알아서 알려주기" — 상시 폴더 감시

사용자가 수신을 시작하는 행위를 제거한다. 서버가 등록된 폴더를 상시 모니터링하고, 새 파일이 감지되면 EMR 화면에 자동으로 반영한다. 특정 화면에 진입하거나 버튼을 클릭할 필요가 없다.

알림은 EMR 상단 메뉴바의 **무선전송 아이콘에 뱃지 숫자**로 표시한다. 토스트나 모달 같은 침습적 알림은 사용하지 않는다. 진료 중 화면을 가리거나 주의를 빼앗는 요소는 진료 흐름을 방해한다. 뱃지는 "볼 때 보는" 수준이면 충분하다 — 사진이 미아가 되지 않는다는 확신만 주면 된다.

### "한 화면에서 선택-편집-추가" — 이미지 뷰어 모달

무선전송 아이콘을 클릭하면 전체 모달이 열린다. 이 모달은 전통적인 이미지 뷰어 형태로 설계했다.

**좌측 — 2컬럼 썸네일 그리드.** 수신된 모든 사진을 최신순으로 나열한다.
추가 완료 사진은 반투명 + 체크 오버레이로 시각적으로 구분하고 있으나 -> 유지/제거 판단해야 함

썸네일 클릭은 **미리보기 전환 + 선택 토글**을 동시에 수행.

각 썸네일의 우하단에 색상 도트 + humanized 시간("8분 전")을 표시한다. 색상 도트는 카메라 소스를 구분하기 위한 것. 이미지 파일 감시 대상이 2개 이상인 경우에만 표시.

**우측 — 미리보기 + 편집 도구.** 선택된 사진을 크게 보여주고, 간단판 편집. 이미 이미지 편집 기능은 "사진보기" 기능에서 구현되어 있음.

**하단 — 대상 환자 선택 + 추가.** "n개 선택됨 [홍길동 ▾] 에 [추가하기]" 구성. 환자명 부분은 클릭하면 위로 펼쳐지는 combobox로, 오늘 접수한 환자 목록을 기본 표시하고, 검색 입력 시 전체 DB에서 환자를 찾을 수 있다. 환자가 잘못 선택된 경우 대체옵션을 제시하지 않으면 지금까지 선택하고 편집한 모든 사항들이 사라짐.

## 4. 프로토타입 설명

### EMR 셸 — 이 기능이 놓이는 맥락

이 모듈은 전자서명처럼 독립된 페이지가 아니다. EMR 진료 화면의 상단 메뉴바에 아이콘 하나가 추가되는 형태이다. 프로토타입에서는 이 맥락을 전달하기 위해 가짜 EMR 셸을 배경으로 깔았다.

상단 바 구성: 좌측에 환자 정보(이름, 차트번호, 생년월일), 가운데에 메뉴 아이콘(차트, 진료, 처방, 영상, 수납, **무선전송**), 우측에 날짜. 환자 정보와 메뉴를 한 줄로 통합한 것은 수직 공간 절약을 위한 것이다.

무선전송 아이콘에 미추가 사진 수가 뱃지로 표시된다. 아이콘을 클릭하면 이미지 뷰어 모달이 열린다.

### 프로토타입 탭 구성

| 탭        | 내용                                                                             |
| --------- | -------------------------------------------------------------------------------- |
| 수신 흐름 | EMR 셸 + 무선전송 아이콘 클릭 → 모달 → 선택 → 추가 → 완료 토스트                 |
| 사진 목록 | 수신된 전체 사진을 날짜/소스별로 그리드 표시. 소스 필터, 추가완료 표시/숨김 토글 |
| 설정      | 감시 폴더 관리, 라벨 색상 지정, 알림 토글                                        |

## 5. 정책 요약

### EMR이 담당하는 범위

| 담당함                          | 담당하지 않음                       |
| ------------------------------- | ----------------------------------- |
| 폴더 감시 → 새 파일 감지 → 알림 | 카메라 → PC 전송 (Eye-Fi, Wi-Fi 등) |
| 사진 미리보기 · 간단 편집       | 본격적인 이미지 편집                |
| 다중 폴더 라벨 · 색상 관리      | 카메라 설정                         |

### 알림 정책

| 항목      | 정책                       |
| --------- | -------------------------- |
| 알림 형태 | 메뉴바 아이콘 뱃지 (숫자)  |
| 알림 대상 | 환자가 선택된 클라이언트만 |
| 비활성화  | PC별 "알림 받지 않기" 토글 |

### 사진 상태

| 상태      | 의미                                        | 시각적 표현            |
| --------- | ------------------------------------------- | ---------------------- |
| 미추가    | 폴더에 도착했으나 환자에 연결되지 않은 사진 | 기본 표시              |
| 추가 완료 | 환자 차트에 연결된 사진                     | 반투명 + 체크 오버레이 |

---

## 6. 기타사항

- **이미지 뷰어 통합**: 현재 EMR에 이미 이미지 뷰어가 존재한다.
- 사진을 추가하기로 하면 모달이 닫혀야 한다.
- 뱃지 숫자 알림 표시: 서버에서 감시하며 모든 클라이언트에 표시
- 뱃지 숫자 알림 초기화: 어느 한 클라이언트에서 1개라도 사진 추가를 하면 0이 되고, 0은 표시하지 않음.

- 모달을 닫았다가 다시 열면 선택/편집 상태가 유지되는가? — 초기화
- 사진이 0장일 때 무선전송 아이콘 클릭하면? - 모달이 열림.
- 모달 하단의 환자 combobox "오늘 접수" 정렬 기준 — 접수 순서(최근 위)
- 사진이 계속 도착하는 중에 모달이 열려 있으면? — 새 사진이 실시간으로 사이드바에 상단으로 추가되어야 함
- 추가 완료 토스트에 "실행 취소" 필요한가? 필요없음
- 썸네일 클릭으로 선택 해제 시 미리보기는? - 빈 상태가 되어야 함

- 추가 완료된 사진 체크박스+오버레이 표시 -> 서버 기준으로, 한번이라도 아무 환자에게나 추가된 사진은 이미 추가한 것으로 표시.
- 위 기능은 .. 트레이드 오프 판단해보고 도입 여부 결정해야 함
