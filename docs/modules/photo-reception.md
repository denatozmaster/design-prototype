# 임상사진 무선 수신

## 1. 업무 배경

### 치과에서 임상사진이란

치과 진료에서 구강 내 사진은 진단·치료계획·경과관찰의 핵심 기록이다. DSLR 카메라로 촬영하며, 환자 한 명당 3~10장, 교정과의 경우 매 방문마다 세트로 촬영한다. 이 사진이 환자 차트에 정확히 붙어야 의미가 있다.

### 현재의 촬영-저장 파이프라인

카메라에서 PC까지의 전송은 EMR이 관여할 영역이 아니다. Eye-Fi, Camfi, 카메라 내장 Wi-Fi 등 다양한 전송 수단이 있지만, 최종 결과는 동일 — **PC의 특정 폴더에 이미지 파일이 생성되는 것**. EMR은 이 폴더만 감시하면 된다.

대부분의 치과(약 90%)는 카메라가 한 대이므로 감시 대상 폴더도 하나다. 다수의 진료실을 운영하는 치과(약 10%)에서만 카메라가 여러 대이고, 그에 따라 감시 폴더도 복수가 된다.

### 현행 시스템(덴트웹)의 문제

덴트웹에서는 사진을 EMR에 추가하려면 **수신대기를 수동으로 시작**해야 한다. 별도 모달 창을 열어서 "수신 시작" 버튼을 누른 뒤, 그 창 안에서 사진을 확인하고 환자에 추가한다.

바쁜 진료 중에 이 과정을 깜빡하면 사진이 폴더에만 존재하고 EMR에서는 인식하지 못한다. 나중에 파일 탐색기나 이미지 뷰어에서 수동으로 찾아 붙여야 하는데, 어떤 사진이 어느 환자 것인지 기억에 의존해야 한다.

근본 원인은 "폴더 감시"라는 단순한 작업을 사용자의 명시적 액션에 의존하게 설계한 것이다.

## 2. 실제 업무 흐름

### 일반과 진료실의 하루

원장이 환자를 진료한다. 진료 중 구강 사진을 찍어야 하는 시점이 오면, 어시스턴트가 DSLR을 건네거나 미러·리트랙터를 잡아준다. 원장이 촬영 후 카메라를 내려놓으면, Wi-Fi를 통해 사진이 PC 폴더에 도착한다. 이 시점에서 원장은 이미 다음 진료 행위에 집중하고 있다.

사진이 폴더에 도착한 건 아무도 모른다. 현행 시스템에서는 진료 후에 EMR에서 수동으로 확인해야 하는데, 다음 환자가 바로 들어오면 그 타이밍을 놓친다. 한두 시간 후에 "아까 찍은 사진이 차트에 안 붙어있네"를 발견하면, 그때서야 폴더를 뒤져서 시간대를 보고 환자를 추정하여 수동으로 연결한다.

### 교정과 진료실의 하루

교정과는 더 빈번하다. 거의 모든 환자마다 교정 진행 상태를 기록하기 위해 정해진 앵글로 세트 촬영을 한다(정면, 좌측, 우측, 상악, 하악 등). 하루에 수십 명의 환자를 보면서 세트마다 5~8장씩 쌓인다. 촬영과 진료가 반복되므로, 사진이 도착하는 즉시 현재 환자에 매칭되는 흐름이 아니면 나중에 일일이 분류하는 작업이 발생한다.

### 다중 카메라 환경 (소수 치과)

진료실이 2~3개인 치과에서는 각 진료실에 카메라가 있고, 각각 다른 폴더로 사진이 전송된다. "이 사진이 어느 진료실에서 온 건지"를 구분할 수 있으면 편하지만, 대다수의 치과에서는 카메라가 한 대이므로 이 구분이 필요 없다. 따라서 다중 소스 구분은 지원하되 UI에서 강조하지 않는다.

## 3. 문제와 설계 의도

### "알아서 알려주기" — 상시 폴더 감시

사용자가 수신을 시작하는 행위를 제거한다. 서버가 등록된 폴더를 상시 모니터링하고, 새 파일이 감지되면 EMR 화면에 자동으로 반영한다. 특정 화면에 진입하거나 버튼을 클릭할 필요가 없다.

알림은 EMR 상단 메뉴바의 **무선전송 아이콘에 뱃지 숫자**로 표시한다. 토스트나 모달 같은 침습적 알림은 사용하지 않는다. 진료 중 화면을 가리거나 주의를 빼앗는 요소는 진료 흐름을 방해한다. 뱃지는 "볼 때 보는" 수준이면 충분하다 — 사진이 미아가 되지 않는다는 확신만 주면 된다.

### "한 화면에서 선택-편집-추가" — 이미지 뷰어 모달

무선전송 아이콘을 클릭하면 전체 모달이 열린다. 이 모달은 전통적인 이미지 뷰어 형태로 설계했다.

**좌측 — 2컬럼 썸네일 그리드.** 수신된 모든 사진을 최신순으로 나열한다. 미추가 사진과 추가 완료 사진을 모두 표시하되, 추가 완료 사진은 반투명 + 체크 오버레이로 시각적으로 구분한다. 처음에는 미추가 사진만 표시하려 했으나, 촬영 세트 전체를 한눈에 보면서 "이건 추가했고, 이건 아직"을 파악하는 것이 빠뜨린 사진을 발견하기에 더 유리하다.

썸네일 클릭은 **미리보기 전환 + 선택 토글**을 동시에 수행한다. 미리보기와 선택을 분리하는 것이 일반적인 패턴이지만, 2컬럼 그리드에서 썸네일 자체가 충분히 크기 때문에 "보기만 하고 선택은 안 하는" 상황이 드물어 합쳐도 무방하다. 체크박스는 선택 상태 표시용으로만 남긴다.

각 썸네일의 우하단에 색상 도트 + humanized 시간("8분 전")을 표시한다. 색상 도트는 카메라 소스를 구분하기 위한 것으로, 텍스트 라벨(예: "Nikon Z50")은 공간을 지나치게 차지하므로 도트만 사용한다.

**우측 — 미리보기 + 편집 도구.** 선택된 사진을 크게 보여주고, 상단에 간단한 편집 아이콘(회전, 크롭, 좌우반전, 상하반전, 밝기, 대비, 색조, 삭제)을 배치한다. 편집 기능의 구현 깊이는 개발팀과 별도 협의 사항이나, 아이콘 배치와 인터랙션 위치는 이 프로토타입이 제안하는 바와 같다.

**하단 — 대상 환자 선택 + 추가.** "n개 선택됨 [홍길동 ▾] 에 [추가하기]" 구성. 환자명 부분은 클릭하면 위로 펼쳐지는 combobox로, 오늘 접수한 환자 목록을 기본 표시하고, 검색 입력 시 전체 DB에서 환자를 찾을 수 있다. 이렇게 한 이유는, 촬영 시점의 현재 환자와 사진을 추가하는 시점의 환자가 다를 수 있기 때문이다. 진료 흐름에서 환자가 바뀐 뒤에야 이전 환자의 사진을 추가하는 상황이 빈번하다.

### 소스별 필터를 넣지 않는 이유

프로토타입 작업 중 소스별 필터 기능의 위치와 필요성을 검토했다. 결론적으로 넣지 않기로 했다. 다중 카메라를 사용하는 치과가 10% 미만이고, 90%의 사용자에게 의미 없는 UI는 노이즈다. 소스 구분이 필요한 경우 색상 도트만으로 시각적 식별이 가능하며, 필터가 반드시 필요한 상황이 생기면 그때 추가해도 늦지 않다.

### 설정 — 최소한의 관리

감시 폴더 등록/삭제, 폴더별 라벨과 색상 지정, 알림 수신 토글. 이 세 가지만 설정에 둔다.

알림 수신 토글은 로컬 서버-다중 클라이언트 구조에서 필요하다. 진료실 PC에서는 알림을 받아야 하지만, 데스크나 상담실 PC에서는 사진 수신 알림이 불필요하다. "이 컴퓨터에서 알림 받지 않기" 한 줄이면 충분하다.

## 4. 프로토타입 설명

### EMR 셸 — 이 기능이 놓이는 맥락

이 모듈은 전자서명처럼 독립된 페이지가 아니다. EMR 진료 화면의 상단 메뉴바에 아이콘 하나가 추가되는 형태이다. 프로토타입에서는 이 맥락을 전달하기 위해 가짜 EMR 셸을 배경으로 깔았다.

상단 바 구성: 좌측에 환자 정보(이름, 차트번호, 생년월일), 가운데에 메뉴 아이콘(차트, 진료, 처방, 영상, 수납, **무선전송**), 우측에 날짜. 환자 정보와 메뉴를 한 줄로 통합한 것은 수직 공간 절약을 위한 것이다.

무선전송 아이콘에 미추가 사진 수가 뱃지로 표시된다. 아이콘을 클릭하면 이미지 뷰어 모달이 열린다.

### 프로토타입 탭 구성

| 탭 | 내용 |
|---|---|
| 수신 흐름 | EMR 셸 + 무선전송 아이콘 클릭 → 모달 → 선택 → 추가 → 완료 토스트 |
| 사진 목록 | 수신된 전체 사진을 날짜/소스별로 그리드 표시. 소스 필터, 추가완료 표시/숨김 토글 |
| 설정 | 감시 폴더 관리, 라벨 색상 지정, 알림 토글 |

## 5. 정책 요약

### EMR이 담당하는 범위

| 담당함 | 담당하지 않음 |
|---|---|
| 폴더 감시 → 새 파일 감지 → 알림 | 카메라 → PC 전송 (Eye-Fi, Wi-Fi 등) |
| 사진 미리보기 · 간단 편집 | 본격적인 이미지 편집 |
| 환자 매칭 · 차트 저장 | 원본 파일 삭제/이동 |
| 다중 폴더 라벨 · 색상 관리 | 카메라 설정 |

### 알림 정책

| 항목 | 정책 |
|---|---|
| 알림 형태 | 메뉴바 아이콘 뱃지 (숫자) |
| 알림 대상 | 환자가 선택된 클라이언트만 |
| 비활성화 | PC별 "알림 받지 않기" 토글 |
| 침습적 알림 (토스트, 모달 등) | 사용하지 않음 |

### 사진 상태

| 상태 | 의미 | 시각적 표현 |
|---|---|---|
| 미추가 | 폴더에 도착했으나 환자에 연결되지 않은 사진 | 기본 표시 |
| 추가 완료 | 환자 차트에 연결된 사진 | 반투명 + 체크 오버레이 |

### 원본 파일 정책

감시 폴더의 원본 파일은 건드리지 않는다. 삭제, 이동, 이름 변경 없음. EMR은 파일을 읽기만 하고, 자체 DB에 사본을 저장한다. 잘못 추가한 경우 기존 이미지 뷰어의 "다른 환자에게 사진 옮기기" 기능으로 대응한다.

## 6. 기타사항

- **이미지 뷰어 통합**: 현재 EMR에 이미 이미지 뷰어가 존재한다. 무선 수신 모달과 기존 뷰어의 관계(통합 vs 병존)는 개발팀과 협의 필요.
- **환자 미선택 상태에서 사진 도착 시**: 미추가 목록에 쌓아두고, 환자가 선택되면 그때 뱃지로 알린다.
- **편집 기능의 구현 범위**: 회전·크롭 정도는 필수, 색조 조정은 우선순위 낮음. 개발팀 판단에 위임.
- **교정과/일반과 분기**: 촬영 빈도와 세트 장수만 다를 뿐 기능은 동일. 별도 분기 불필요. 라벨로 구분 가능.
