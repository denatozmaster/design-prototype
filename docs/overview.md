## 프로젝트 개요

### 무엇을 만드는가

치과 EMR(전자차트) 앱 Dops의 **디자인 프로토타입**. 새로 합류한 디자이너에게 전달하기 위한 참고 자료이다.

### 왜 만드는가

디자이너가 교체되었다. 새 디자이너는 도메인 지식이 없다. CEO(치과의사)가 직접 프로토타입을 만들어, end user 입장에서 화면 구성이 왜 이래야 하는지를 전달하려는 것이다. 디자이너는 이 프로토타입과 문서를 보고 업무 흐름을 체득한 뒤, 더 나은 결과물을 만들 수 있어야 한다.

### 프로토타입의 성격

- DB 연결 없음. 모든 데이터는 가짜로 생성
- 실제 동작이 아닌, 화면 구성과 업무 흐름 전달이 목적
- 세부 디자인(색상, 폰트, 여백 등)은 디자이너가 할 일. 프로토타입은 구조와 의도만 전달
- 디자인 톤은 Vercel풍 흑백 기조에 포인트 컬러만 사용
- 화면 밀도는 높은 편 (전문 EMR 앱 기준)

### 구조 원칙

**각 페이지는 완전히 독립적이다.**

- 랜딩 페이지에서 각 개별 페이지 링크로 진입하는 구조
- 페이지 간 유기적 연결, 공유 상태, 통합 네비게이션 없음
- 각 페이지 = 하나의 기능 모듈 = 하나의 독립된 의도 전달 단위

이 구조를 채택한 이유:

- **원하는 것**: 각 개별 기능에 대한 명확한 의도 전달
- **원하지 않는 것**: 기능들을 묶어서 전체를 유기적으로 연결하는 것. 이건 오버엔지니어링
- 모듈별로 순서에 구애받지 않고, 급한 것부터 하나씩 만들어 쌓아갈 수 있음
- 디자이너도 모듈 하나씩 소화 가능

### 각 모듈의 구성

모듈 하나는 **문서 + 프로토타입** 한 쌍으로 이루어진다.

- **문서**: 업무 배경 → 실제 업무 흐름 → 문제와 설계 의도 → 프로토타입 설명 → 정책 요약 (create_docs.md 원칙 참고)
- **프로토타입**: 문서의 내용을 시각적으로 구현한 화면

### 배포

GitHub Pages로 팀원(개발팀 6인 + 디자이너)에게 공유. GitHub Enterprise 사용 중이므로 private repo에서 팀원에게만 접근 제한 가능.

### 개발환경

- windows 10/11
- powershell
- nodejs 22.4

### 기술 스택

- Vite + React 19 + TypeScript
- TailwindCSS v4 + shadcn-ui
- React Router (랜딩 → 개별 페이지 진입 용도로만)
- gh-pages 배포

### 팀 구성

- CEO (이 프로토타입의 작성자): 치과의사, 도메인 전문가, 개발은 취미 수준
- 전문 개발팀 6인
- 디자이너 1인 (신규 합류, 도메인 지식 없음)
