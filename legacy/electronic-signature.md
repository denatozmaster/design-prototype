## 1. 업무 배경

### 왜 전자서명이 필요한가

자필로 작성된 기록은 작성 즉시 법적 효력을 갖지만, 전자기록은 전자서명을 해야만 법적 효력이 발생한다. 전자서명이 없는 문서는 법적으로 효력이 없다.

치과 병원은 의무기록을 작성하고 서명할 의무가 있다. 전자차트 환경에서는 기록은 실시간으로 쌓이지만 서명은 별도 행위이다 보니, "나중에 몰아서 하는" 패턴이 자리잡았다. 서명을 미루면 수백~수천 건이 쌓이고, 감사나 점검 시 미서명 기록이 발견되면 행정 처분의 대상이 된다.

### 3종 문서의 성격

**진료기록부**는 환자를 진료할 때마다 생성된다. 하루에 수십~수백 건이 쌓이고, 진료한 의사 각자가 본인 기록에 서명해야 한다.

**본인부담금 수납대장**과 **방사선발생장치 사용대장**은 병원 단위로 하루에 각각 1건씩 생성된다. 병원 별로 생성되기에 대표원장만 서명한다.

### 병원 내 역할 구조

**대표원장**은 병원의 개설자이자 최종 책임자이다. 위 3종류의 문서에 서명 의무가 있고, 원내 모든 의사의 서명 현황을 관리해야 한다.

- **페이닥터(봉직의)**는 고용된 진료 의사. 본인이 진료한 기록에만 서명하면 된다.
- **중간관리자(실장)**는 직접 서명하지는 않지만, 원내 서명 현황과 서명 이력을 확인하여 대표원장에게 보고하거나 봉직의에게 서명을 독려하는 역할.

> 이 문서와 제공된 프로토타입 디자인은 **대표원장 화면**을 다룬다. 페이닥터 및 중간관리자 화면은 대표원장 화면에서 역할에 필요한 부분만 발췌하여 구성

## 2. 실제 업무 흐름

### 일상적인 서명 패턴

**패턴 A(70%) — 월 1회 몰아서 서명.** 월초에 지난달 진료기록 전체(3~4천 건)를 한번에 서명. 수납대장과 방사선대장도 이때 같이 처리. 가장 흔한 시나리오이고, **일괄서명** 기능이 이 패턴을 위해 존재한다.

**패턴 B(30%) — 3일 이내 수시 서명.** 차트를 기록한 뒤 당일 혹은 며칠 내에 서명.

### 페이닥터 서명 관리

페이닥터는 병원에서 일을 하지만 길지 않은 시간만 함께하는 객식구 같은 존재이다. 본인의 진료 외의 다른 부분은 별 관심이 없다. 페이닥터가 서명을 안 하고 미루면 대표원장이나 실장이 관리해야 한다. 대시보드에서 각 페이닥터의 마지막 서명일과 미서명 건수를 한눈에 확인할 수 있어야 하는 이유.

### 다른 의사 기록에 서명하는 경우

원칙적으로 진료기록부 서명은 진료를 수행한 의사 본인이 해야 한다. 그러나 현실에서는 페이닥터가 퇴사한 뒤 연락이 두절되는 경우가 있다. 이때 미서명 기록이 수백 건 남아 있으면, 대표원장이 이를 처리할 수밖에 없다.

이 상황은 법적 근거가 명확하지 않은 회색 지대이다. 그래서 UI에서 "대리서명"이라는 용어를 의도적으로 사용하지 않고, 대신 중립적 표현. 필요한 기능은 제공하되 법적 리스크가 될 수 있는 표현은 피한다. 내부적으로는 누가 서명했는지 이력을 남겨 사실관계를 기록.

### 공인인증서

전자서명은 공인인증서 기반으로, 인증서는 서버에 저장되고, PIN 6자리로 인증한다. 인증서 등록/갱신은 별도 설정 페이지에서 관리하며, 이 섹션에서는 상태 요약과 설정 페이지 링크만 제공.

## 3. 이런 업무 흐름에서 발생하는 문제와 설계 의도

### "서명을 미루는 것"이 핵심 문제

서명 자체는 PIN 입력 한번이면 끝. 문제는 "언제 해야 하는지 잊는 것"과 "쌓이면 귀찮아지는 것"이다. 설계 전반에서 이 두 가지를 해결하려고 한다.

**잊지 않게 하기:**

- 사이드바 메뉴에 경과일수를 표시. 한달을 초과하여 미서명인 문서가 있으면 해당 메뉴 옆에 빨간 숫자(예: "44일")
- 대시보드에서 마지막 서명일과 경과일수를 크게 보여준다. 미서명 건수보다 "얼마나 오래 안 했는지"가 행동을 유발하는 더 강한 동기이다.

**쌓여도 빠르게 처리하게 하기:**

- 일괄서명은 기간 선택 → 조회 → PIN 한번으로 3종 문서 수천 건을 처리하게 된다. 서버에서 처리하므로 사용자 화면에서는 백그라운드에서 진행 상황을 별도로 feedback 하여, 서명하는 동안 다른 작업을 할 수 있도록 해야 한다.
- 기간 선택 퀵 버튼을 "최근 3일 / 지난주 / 지난달 / 이번달 / 전체"로 구성한 이유는, 실제 사용 패턴이 "다음달 초에 지난달 것을 몰아서"이기 때문이다.

### 서명 행위는 최소한의 단계로

PIN 6자리를 입력하면 별도 "확인" 버튼 없이 자동으로 서명이 실행된다. PIN 입력 자체가 서명 의사의 확인이므로 추가 클릭은 불필요. 서명 완료 후 모달은 즉시 닫히고, 우측 상단 토스트로 처리 상태를 보여준다.

## 4. 프로토타입 설명

### 메뉴 구성 (6개)

| 메뉴       | 대응하는 업무 시나리오                                           |
| ---------- | ---------------------------------------------------------------- |
| 서명 현황  | 로그인 후 "지금 뭐가 밀려있지?" 확인하고 바로 서명 작업으로 진입 |
| 일괄서명   | 월초에 지난달 것을 3종 문서 한번에 처리                          |
| 진료기록부 | 진료기록을 환자별로 확인하며 서명. 필요 시 다른 의사 기록도 처리 |
| 수납대장   | 수납대장 날짜별 확인 및 서명                                     |
| 방사선대장 | 방사선대장 날짜별 확인 및 서명                                   |
| 서명 이력  | "지난달에 언제 서명했더라?" 확인. 감사/점검 대비                 |

### 화면별 주요 포인트

**서명 현황**: 마지막 서명일 + 경과일수가 가장 크게 노출. 미서명 건수 카드는 각각 클릭 가능하여 해당 서명 페이지로 링크. 전체 미서명 카드는 일괄서명으로 연결되며, 같은 카드 그룹 안에서 은은하게 구분(blue 테두리 + blue 숫자). 하단에 의사별 서명 현황 테이블이 있어 페이닥터의 서명 상태 파악 가능.

**일괄서명**: 기간 선택 → 조회 → 3종 문서 카드(토글) → 서명하기 → PIN → 토스트. 가장 빈번한 시나리오

**진료기록부**: 기간 선택 + "모든 의사 기록 검색" 체크박스 → 조회(5초 로딩 프로그레스) → 환자별 목록(의사별 그룹핑) → 건별 선택/해제 → PIN.

**수납대장 / 방사선대장**: 기간 선택 → 조회 → 날짜별 목록 → 선택 → PIN.

**서명 이력**: 기간 선택(기본값 1년) → 전체 서명 행위 목록. 서명일시, 서명자, 건수, 문서 종류, 특이사항을 표시합니다. 필요한 경우 서명자 필터 제공. 이 페이지에서 궁금한 것은 “누가”서명을 했느냐 이지 진료를 누가 했느냐가 아님.

## 5. 정책 요약

### 상태 정의

| 상태     | 조건                                 | 표시       |
| -------- | ------------------------------------ | ---------- |
| 서명필요 | 미서명 또는 서명 후 내용이 수정된 건 | amber 뱃지 |
| 서명완료 | 서명이 완료되고 내용 변경 없는 건    | green 뱃지 |

덴트웹의 미서명과 재서명(서명 후 수정)을 "서명필요"로 통합. 사용자에게 중요한 것은 "지금 서명해야 하는가" 여부

### 권한 매트릭스

| 기능                 | 대표원장                | 봉직의       | 중간관리자 |
| -------------------- | ----------------------- | ------------ | ---------- |
| 서명 현황 (대시보드) | 전체 조회               | 본인만       | 전체 조회  |
| 일괄서명             | 3종 전체                | 진료기록부만 | ✕          |
| 진료기록부 서명      | ○ + 모든 의사 기록 검색 | ○ (본인만)   | ✕          |
| 수납대장 서명        | ○                       | ✕            | ✕          |
| 방사선대장 서명      | ○                       | ✕            | ✕          |
| 서명 이력            | 전체 조회               | 본인만       | 전체 조회  |

### 기간 선택 퀵 버튼

모든 기간 선택 UI에 동일하게 적용: **최근 3일 / 지난주 / 지난달 / 이번달 / 전체**

"전체"는 개원시작일 기준으로, 개원시작일은 별도 변수로 앱 전체 레벨에서 관리되어야 함. 현재 선택된 날짜 범위가 퀵 버튼과 정확히 일치하면 해당 버튼이 active 상태로 표시. 기간 선택은 가장 최근 사용한 퀵 버튼 종류를 기본값으로 설정.

### 메뉴 경고 표시 규칙

진료기록부, 수납대장, 방사선대장 메뉴에 한해, 마지막 서명일로부터 30일 이상 경과 시 메뉴 오른쪽에 경과일수를 빨간색으로 표시합니다. 30일 미만이면 표시하지 않습니다.

### 서명 처리 흐름

PIN 6자리 완성 → 모달 즉시 닫힘 → 우측 상단 토스트(프로그레스 바) → 논블로킹으로 다른 작업 가능 → 완료 시 성공 표시 후 3초 뒤 자동 제거

## 6. 기타사항

- **서명 취소/철회**: 불필요한 기능.
- **로그인 시 알림**: 미서명 1개월 이상 경과 시 로그인 직후 feedback 필요. (로그인은 매일 1회만 발생. 일정기간 보지 않기 옵션이 필요하다면 1주일 정도가 적당할듯)
