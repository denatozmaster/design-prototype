# 임상사진 무선 수신 — 프로토타입 구체화 계획

## 1. 이 모듈이 전달해야 하는 것

현행 시스템의 문제는 "사진 수신을 수동으로 시작해야 한다"는 것. DSLR로 촬영하면 Wi-Fi를 통해 PC 폴더에 사진이 떨어지는데, EMR에서 그걸 인식하려면 사용자가 별도 모달을 열어야 한다. 바쁜 진료 중에 그걸 깜빡하면 사진이 미아가 된다.

프로토타입이 보여줘야 하는 핵심:
- **사용자가 아무것도 안 해도** 사진이 도착하면 알아서 알려주고, 환자에 붙여주는 흐름
- 진료 화면을 가리지 않는 비침습적 알림
- 다중 카메라/폴더를 라벨로 구분하는 설정 화면

## 2. 프로토타입의 맥락 — EMR 안에서의 위치

이 기능은 전자서명처럼 독립된 페이지가 아니다. 실제 EMR에서는 진료 화면 위에 떠있는 패널/알림이다. 프로토타입에서는 이 맥락을 전달하기 위해 **가짜 EMR 셸(진료 화면 배경)**을 깔고 그 위에서 사진 수신 흐름을 시연한다.

가짜 EMR 셸 구성:
- 상단 바: 환자 정보 (이름, 차트번호, 생년월일)
- 좌측: 간략한 차트 영역 (실제 데이터 불필요, 배경 역할)
- 우측 하단: 사진 수신 패널이 위치할 영역

## 3. 화면 구성 (3개 탭)

### 탭 A. 수신 흐름 시연

사진이 도착 → 알림 → 확인 → 저장까지의 흐름을 단계별로 보여준다. 자동 진행 또는 버튼 클릭으로 다음 단계로 넘어가는 시뮬레이션.

**단계 1: 대기 상태**
- EMR 셸 배경, 환자 "홍길동" 선택된 상태
- 우측 하단에 사진 수신 아이콘 (접힌 상태, 뱃지 없음)

**단계 2: 사진 도착 알림**
- 토스트 알림: "Nikon Z50에서 새 사진 2장"
- 수신 아이콘에 뱃지 숫자 표시 (2)
- 알림은 진료 화면을 가리지 않음 — 우측 상단 토스트

**단계 3: 사진 확인 패널 열기**
- 토스트 클릭 또는 수신 아이콘 클릭으로 패널 오픈
- 패널 내용:
  - 현재 환자: "홍길동 (2024-00123)" 표시
  - 새로 도착한 사진 썸네일 (2장)
  - 각 사진의 촬영 시간, 소스 폴더 라벨
  - "환자에 추가" 버튼

**단계 4: 저장 완료**
- "환자에 추가" 클릭 → 성공 피드백
- 사진이 "추가 완료" 상태로 전환 (체크 표시, 흐리게)
- 뱃지 숫자 사라짐

### 탭 B. 사진 목록

수신된 사진의 전체 목록을 보여주는 화면. 실제 사용에서는 패널 내 탭 또는 확장 뷰.

- **미추가 사진**: 기본 표시, 썸네일 + 촬영시간 + 소스 라벨
- **추가 완료 사진**: 기본 숨김, 필터 토글로 표시 가능
- 각 사진에 매칭된 환자명 표시 (추가 완료 시)
- 날짜/소스별 그룹핑

### 탭 C. 설정

감시 폴더 관리 화면.

- 등록된 폴더 목록 (테이블)
  - 폴더 경로
  - 라벨 (편집 가능)
  - 감시 상태 (활성/비활성)
  - 최근 수신 시간
- 폴더 추가 다이얼로그: 경로 입력 + 라벨 + 확장자 필터 (jpg, png, raw 등 체크박스)
- 알림 설정: "이 컴퓨터에서 알림 받지 않기" 토글
- 기본 저장 카테고리 설정 (구강사진, 방사선사진 등 — 폴더별 지정 가능)

## 6. 전자서명 모듈과의 차이점

| 항목 | 전자서명 | 임상사진 수신 |
|---|---|---|
| 레이아웃 | 사이드바 + 콘텐츠 | 가짜 EMR 셸 + 오버레이 패널 |
| 성격 | 독립된 관리 페이지 | EMR 진료 화면에 내장된 기능 |
| 핵심 전달 | 서명 업무 흐름 | 알림 → 확인 → 저장의 자연스러운 흐름 |
| 인터랙션 | 조회 → 선택 → PIN | 자동 알림 → 확인 → 추가 |

## 7. 프로토타입에서 다루지 않는 것

- 실제 폴더 감시 동작 (서버/Electron 레벨 — 개발팀 영역)
- 이미지 뷰어 통합 (미결 사항으로 남겨둠)
- 교정과/일반과 분기 (본질은 동일, 라벨로 구분 가능)
- 사진 삭제/이동 (원본 파일은 건드리지 않는다는 정책)
- "다른 환자에게 사진 옮기기" (기존 이미지 뷰어 기능, 이 모듈 범위 밖)
