# 급여레진 치아면 선택 — 구현 계획

## 모듈 위치

`src/pages/resin-surface/` — clinical-chart 템플릿과 무관한 독립 페이지

## 화면 구성

좌우 2단 레이아웃. 실제 앱에서는 모달이지만 프로토타입에서는 페이지로 펼쳐놓는다.

### 좌측: 치아 선택 + 면 선택 영역

**상단 — 치식(dental chart)**
- 성인 치식 32개 치아를 상악/하악 좌우로 배치 (FDI 표기: 11~18, 21~28, 31~38, 41~48)
- 치아번호 클릭 → 하단 면 선택 영역에 해당 치아 카드가 추가됨
- 다시 클릭 → 해당 치아 카드 제거
- 선택된 치아는 치식에서 활성 스타일로 표시

**하단 — 면 선택 카드 목록**
- 선택된 치아마다 카드 1개가 생성되어 나란히 배치
- 각 카드 구성:
  - 치아번호 + 전치부/구치부 표시
  - 치아 도식: 십자형 배치 (상B, 좌M, 중O, 우D, 하L). legacy 배치를 따름
  - 면 버튼 클릭으로 토글. 선택된 면은 채색, 미선택은 비활성
  - 급수: 자동 계산값 표시 + 드롭다운으로 수동 오버라이드 가능(1~6급)
  - 면수: 선택된 면 개수. 자동 계산, 오버라이드 불가

### 우측: 보험코드 자동 연동 영역

좌측의 면 선택 결과가 실시간으로 반영된다.

- **치아별 코드 매핑**: 각 치아가 어떤 보험코드에 해당하는지 개별 표시
  - #46: 3면 → U0241
  - #11: 1면 → U0239
  - 등
- **코드 요약**: 
  - U0239 광중합형 복합레진 충전-1면 × n
  - U0240 광중합형 복합레진 충전-2면 × n
  - U0241 광중합형 복합레진 충전-3면이상 × n
- 면이 미선택(0면)인 치아는 코드 미표시
- 면 선택이 변경되면 즉시 갱신

### 하단: 확인 버튼

## 자동 계산 로직

task/overview.md의 규칙을 그대로 구현한다.

- 전치부/구치부는 치아번호로 판별
- 면수 = 선택된 면 개수
- 급수 = 전치부/구치부별 면 조합 규칙
- 수동 오버라이드: 급수 드롭다운 변경 시 유지, 면 재변경 시 자동 복귀
