DSLR 촬영 (교정과)
본질
EMR이 담당하는 영역은 로컬 폴더 감시 → 알림 → 환자 매칭 → DB 저장이라는 단순한 파이프라인이다. 전송 수단(Eye-Fi, Camfi, 카메라 내장 Wi-Fi 등)과 무관하게 최종 결과는 동일 — PC의 특정 폴더에 이미지 파일이 생성되는 것. EMR은 이 폴더만 감시하면 된다.
현행 문제 (덴트웹)

- 수신대기를 수동 시작해야 함 — 바쁜 진료 중 깜빡하면 사진이 폴더에만 존재하고 EMR에서 인식 불가. 이후 이미지 뷰어에서 파일을 찾아서 다시 추가해야 함
- 별도 모달 창 — 진료 화면 위에 덮여 컨텍스트 전환 발생
  근본 원인: "폴더 감시"라는 단순한 작업을 사용자의 명시적 액션에 의존하게 설계한 것.
  요구사항
  상시 폴더 감시
- 서버가 등록된 폴더를 상시 모니터링. 사용자 액션 불필요.
- 서버가 실행 중 항상 동작. 특정 화면 진입이나 버튼 클릭과 무관.
  알림
- 라벨 + 새 사진 수 표시 (예: "Nikon Z50에서 새 사진 2장")
- 환자가 선택된 클라이언트에만 feedback (로컬 서버-다중 클라이언트 구조)
  - 진료실이 아닌 데스크, 상담실의 경우 “이 컴퓨터에서 알림 받지 않기” 등의 옵션 필요
- 진료 흐름을 방해하지 않는 비침습적 형태
  환자 매칭 및 저장
- 현재 선택된(메인) 환자에 자동 매칭
- 사용자 확인 후 저장
- 잘못 저장한 경우 기존 이미지 뷰어의 "다른 환자에게 사진 옮기기" 기능으로 대응
  다중 폴더 + 라벨
- 감시 폴더를 복수 등록 가능 (설정 화면에서 관리)
- 각 폴더에 사용자 지정 라벨 부여 (예: "Nikon Z50", "진료실2")
  사진 목록 관리
- 미추가 사진: 목록에 표시
- 추가 완료 사진: 기본 숨김, 필터로 재확인 가능
- 원본 파일은 건드리지 않음 (감시 폴더의 파일 삭제/이동 없음)
  설정
- 감시 폴더 경로 추가/삭제/수정
- 폴더별 라벨 지정/변경
- 감시 대상 확장자 필터 (jpg, png, raw 등)
  미결 사항
- 알림 클릭 후 사진 확인 UI — 기존 이미지 뷰어와 통합할지, 현행 덴트웹 방식(별도 목록)을 유지할지 논의 필요
