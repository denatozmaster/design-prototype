모듈 이름 : 급여레진 치아면 선택

# 소아 레진 보험청구 — 면·급수·면수 자동 계산

## 현재 문제

### 1. 불필요한 반복 입력

치아의 면(B, M, O, D, L)을 마우스로 클릭한 뒤, 급수(Class)와 면수를 **또다시 수동 선택**해야 한다. 면 정보에 급수·면수가 내포되어 있으므로 이중 입력이다.

### 2. 청구 코드와 면수의 불일치 시 전체 재입력

소아 복합레진 보험청구는 1면/2면/3면이상 코드가 분리되어 있다. 현재 흐름은 **청구 버튼 선택 시점에 면수를 미리 결정**해야 한다. 이후 실제 면 입력 과정에서 면수가 바뀌거나 혼동이 생기면, 해당 항목을 삭제하고 처음부터 재입력해야 한다. 또한 면수가 다른 소아레진 충전을 한 번에 입력할 수 없고, 건별로 따로 입력해야 한다.

면 선택에서 면수가 자동 계산되면, 청구 코드를 면수에 맞춰 자동 연동할 수 있고, 면수 변경 시에도 코드가 따라가므로 삭제·재입력이 불필요해진다.

## 템플릿 화면 구성

좌/우 2단 구성
좌측: 실제 입력용 모달로 열릴 창의 내부를 모달이 아니라 page에 구현
우측: 적용될 보험 코드, 횟수

U0239: 광중합형 복합레진 충전-1면
U0240: 광중합형 복합레진 충전-2면
U0241: 광중합형 복합레진 충전-3면이상

## 자동 계산 로직

### 치아 위치 판별

치아번호로 자동 구분한다.

- **전치부** — #11~13, #21~23, #31~33, #41~43 (중절치~견치)
- **구치부** — #14~18, #24~28, #34~38, #44~48 (소구치~대구치)

### 면수

선택된 면의 개수. B + O + D 선택 → 3면.

### 급수 (Class)

#### 구치부

| 조건                         | 급수 |
| ---------------------------- | ---- |
| M, D 미포함 (O, B, L의 조합) | 1급  |
| M 또는 D 포함                | 2급  |

#### 전치부

전치부에는 교합면이 없다. UI상 O 버튼은 그대로 두되, **내부적으로 절단면(Incisal)으로 취급**한다.

| 조건                        | 급수 |
| --------------------------- | ---- |
| B 또는 L만 (M, D, O 미포함) | 1급  |
| M 또는 D 포함, O 미포함     | 3급  |
| M 또는 D 포함, O 포함       | 4급  |

#### 5급·6급

면 조합으로 판별할 수 없다(치경부 수복, 교두정/절단연단 수복). 드롭다운에 선택지로 존재하지만 **자동 계산 대상에서 제외**. 사용자가 수동으로 선택한다.

---

## 동작 흐름

```
치아번호 선택 (기존 동작, 이미 완료된 상태)
       ↓
면 클릭/해제 (B, M, O, D, L)
       ↓
급수·면수 즉시 자동 반영
       ↓
면수에 따라 보험 청구 코드(1면/2면/3면이상) 자동 연동
       ↓
(필요 시) 사용자가 급수 드롭다운 수동 변경 → 오버라이드
       ↓
면을 다시 변경하면 → 자동 계산으로 복귀
```

---

## 수동 오버라이드 규칙

- 급수 드롭다운은 항상 편집 가능 (1~6급)
- 사용자가 급수를 수동 변경하면, 해당 값을 유지
- 이후 면 선택을 변경하면, 자동 계산이 다시 작동하여 오버라이드 해제
- 면수는 선택된 면의 수이므로 오버라이드 대상 아님
